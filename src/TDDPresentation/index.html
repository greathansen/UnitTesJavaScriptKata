ATAG - TDD (Test Driven Divelopment)


What is Tdd?


  • Test Driven Development is a process, an apporach to develop software.
  • Consist in first make a test, make it's fail, and then create a peace of code that accomplish that test.
  • Then you have to refactor the code to be according to the software development standarts.
  • With this we have an automated test case.

Add a Test and Run Test


In TDD you have firts to add a Test. Is will fail, there is no code that cover that test, but should have the minor steps for reproduce a user case. The test should be descriptive and clean. What does clean means... should contains the minor peace of code to test a user story or user case, and should not to add business logic.

Write some code


Once the test is written, the new feature should be added, so we add some code to add the business logic The code will be not perfect and not the final, but the test should pass.

Refactor the Code


We created a test and we added business logic. Now we have to clean the code and improve our code. and then we re run the test. The test shoul be pass (we dont have to add any code to the test). If the test fails, we have to review if we broke the functionality or if we miss something.

Repeat


TDD is an iterative, so we have to do all the steps for each peace of code that we want to add.

Remember...Keep the test small.
